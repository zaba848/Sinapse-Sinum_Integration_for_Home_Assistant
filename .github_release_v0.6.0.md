## ✨ Highlights

- **Exponential Backoff for WebSocket** — Reconnection now uses intelligent delays (5 → 10 → 20 → 40 → 60s max) instead of fixed 5s. Prevents connection storms when hub restarts or network flaps.
- **WebSocket Enabled by Default** — New installations automatically use real-time transport with <1s latency instead of 30s REST polling
- **Debug Logging for WebSocket** — Unhandled event types now logged at DEBUG level for observability and passive detection of new WS payload types
- **Security: IP Removal** — Removed all 5 production hub IP addresses from repository and documentation (URLs now via environment variables only)
- **HACS Compliance** — Added `homeassistant: 2024.1.0` field for HACS integration and visibility

## 📖 Installation & Upgrade

**New installations**: WebSocket is enabled by default in v0.6.0+

**Existing installations**: On coordinator restart, WS will be enabled automatically. Manual toggle in Settings → Devices & Services → Sinum → Options if needed.

**No breaking changes** — all automations, scenes, and entities continue to work unchanged.

### Quick Start
```
1. HACS → Integrations → Custom repositories
   URL: https://github.com/zaba848/sinapse-sinum-integration-for-home-assistant
   Category: Integration

2. Install "Sinum (Sinapse)" → Restart Home Assistant

3. Settings → Devices & Services → Add Integration → search "Sinum"
   Enter hub IP and API token

4. WebSocket real-time transport is enabled by default. Done.
```
